# Scale-Offset codec

Defines a `array -> array` codec that encodes an array in a new data type after applying an offset, 
followed by a scaling, to the array values. This transformation is commonly used to map floating point values into the support range of an integer data type, which achieves a lossy form of compression. 

## Codec metadata

This codec is declared in metadata as a JSON object with the following structure:

| Field | Type | Required |
| -     | -    | -        |
| [`name`](#name)  | string | yes |
| [`configuration`](#configuration) | object | yes |   

### Name

When writing metadata documents, the value of the `name` field MUST be `scale-offset`.

When reading metadata documents, the value of the `name` field MAY be either `"scale-offset"` or `"numcodecs.fixedscaleoffset"`. The `"numcodecs.fixedscaleoffset"` alias was used by earlier versions of this codec.

### Configuration

The `configuration` field in codec metadata is a JSON object with the following structure:

| Field | Type | Required |
| -     | -    | -        |
| [`offset`](#offset)  | JSON-encoded array scalar | yes |
| [`scale`](#scale) | JSON-encoded array scalar | yes |
| [`dtype`](#dtype) | Zarr V3 data type metadata | yes |
| [`astype`](#astype) | Zarr V3 data type metadata | yes |

Additional keys are reserved for future versions of this codec, and SHOULD be treated as an error by readers.

#### offset

The `offset` field is an array scalar that defines the quantity added to input values during encoding. If `astype` is defined, then `offset` MUST be encoded to JSON using the Zarr V3 fill value encoding for `astype`. If `astype` is not defined, then `offset` MUST be a JSON number.

### scale

The `scale` field is an array scalar that defines the quantity multiplied with input values during encoding, after they have been added to `offset`. If `astype` is defined, then `scale` MUST be encoded to JSON using the Zarr V3 fill value encoding for `astype`. If `astype` is not defined, then `scale` MUST be a JSON number.

### dtype

The `dtype` field defines the data type of the array returned by the encoding procedure, as well as the data type of the array provided as input for the decoding procedure.

When reading metadata, `dtype` MAY be either Zarr V3 or Zarr V2 data type metadata. Zarr V2 data type metadata is permitted for backwards compatibiliy. 

If `dtype` is encoded using Zarr V2 metadata, a reader attemping to encode chunks SHOULD make a best-effort attempt to interpret Zarr V2 data type metadata as Zarr V3 data type metadata. Failing this, a reader SHOULD error rather than encode chunks ambiguously.

When writing metadata, `dtype` MUST be Zarr V3 data type metadata.

### astype

The `astype` field defines the data type of the array provided as input to the encoding procedure, as well as the data type of the array returned by the decoding procedure.

When reading metadata, `astype` MAY be either Zarr V3 or Zarr V2 data type metadata. Zarr V2 data type metadata is permitted for backwards compatibiliy. 

If `astype` is encoded using Zarr V2 metadata, a reader attemping to decode chunks SHOULD make a best-effort attempt to interpret Zarr V2 data type metadata as Zarr V3 data type metadata. Failing this, a reader SHOULD error rather than decode chunks ambiguously.

When writing metadata, `astype` MUST be Zarr V3 data type metadata. 

## Algorithm

### Encoding

Given an input array `I`, an output array `O` with data type `dtype` given by `configuration.dtype` is generated by the following procedure:

`O = cast((I + offset) * scale, dtype)`

### Decoding

The decoding procedure inverts the encoding procedure:

`O_decode = (cast(I_decode, astype) / scale) - offset`

Note that the data type denoted by `astype` must support multiplication, division, addition, and subtraction.

## Example

The following snippet of array metadata demonstrates the metadata for the `scale-offset` codec.

```json
{
    "data_type": "float32",
    "codecs": [
        {
        "name": "scale-offset",
        "configuration": {
            "offset": 5,
            "scale": 10,
            "dtype": "int32",
            "astype": "float32" 
            }
        }, 
        "bytes"
    ],
}
```

## Caveats

This codec relies on converting arrays from one data type to another. Data type conversion is a complex topic, and routines for converting data types often take parameters which are not presently modeled by this codec. Users of this codec should ensure that readers and writers agree on data type conversion behavior before relying on consistency between those readers and writers.

## References

This codec is based on the `FixedScaleOffset` class defined in numcodecs. See https://numcodecs.readthedocs.io/en/stable/filter/fixedscaleoffset.html for details about that class.
