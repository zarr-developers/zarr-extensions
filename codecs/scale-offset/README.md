# Scale-Offset codec

Defines an `array -> array` codec that subtracts an offset, then multiplies by a scaling factor. This transformation is commonly used to map floating-point values into the support range of an integer data type, which achieves a lossy form of compression.

## Codec metadata

This codec is declared in metadata as a JSON object with the following structure:

| Field | Type | Required |
| -     | -    | -        |
| [`name`](#name)  | string | yes |
| [`configuration`](#configuration) | object | yes |

### Name

The value of the `name` field MUST be the string `"scale-offset"`.

### Configuration

The value of the `configuration` field is a JSON object with the following structure:

| Field | Type | Required |
| -     | -    | -        |
| [`offset`](#offset)  | JSON-encoded scalar | yes |
| [`scale`](#scale) | JSON-encoded scalar | yes |

Additional keys are reserved for future versions of this codec. Metadata with additional keys MUST be treated as invalid by readers.

### offset

The value of the `offset` field is a scalar that defines the quantity subtracted from input values during encoding. This value MUST be a JSON number.

### scale

The value of the `scale` field is a scalar that defines the quantity multiplied with input values during encoding, after they have been added to `offset`. This value MUST be a JSON number.


## Algorithm

### Encoding

Given an input array `in`, an output array `out` is generated by the following routine:

```
out = (in - offset) * scale

```

### Decoding

The decoding routine inverts the encoding routine, i.e.:

```
out = (in / scale) * offset
```

## Example

The following snippet of array metadata demonstrates the metadata for the `scale-offset` codec.

```json
{
    "data_type": "float32",
    "codecs": [
        {
            "name": "scale-offset",
            "configuration": {
                "offset": 5,
                "scale": 0.1,
            }
        },
        "bytes"
    ]
}
```

## Implementation notes

To achieve data compression, this codec should be followed by an `array->array` codec that converts arrays to a narrower data type.

## Legacy `numcodecs.fixedscaleoffset` codec

The `scale-offset` codec described here is based on a Zarr V3 codec called `numcodecs.fixedscaleoffset`, which was defined operationally as a direct translation of the Zarr-V2-compatible `fixedscaleoffset` codec into Zarr V3. That `fixedscaleoffset` codec was itself defined operationally by an implementation (`numcodecs.FixedScaleOffset`) in the Numcodecs library.

In conjunction with the `astype` codec, the `scale-offset` codec supersedes the `numcodecs.fixedscaleoffset` codec. Authors of implementations of the `scale-offset` codec may wish to support the `numcodecs.fixedscaleoffset` codec, e.g. by treating it as a read-only alias for a `[scale-offset, astype]` codec sequence. Thus the `numcodecs.fixedscaleoffset` codec will be defined here, with instructions for mapping the semantics of that codec to the `scale-offset` codec.

### Codec metadata

| Field | Type | Required |
| -     | -    | -        |
| [`name`](#name)  | string | yes |
| [`configuration`](#configuration) | object | yes |

### Name

The value of the `name` field MUST be the string `"numcodecs.fixedscaleoffset"`.

### Configuration

The value of the `configuration` field is a JSON object with the following structure:

| Field | Type | Required |
| -     | -    | -        |
| [`offset`](#offset-1)  | number | yes |
| [`scale`](#scale-1) | number | yes |
| [`astype`](#astype-1) | Zarr V2 data type metadata | yes |
| [`dtype`](#dtype-1) | Zarr V2 data type metadata | yes |

#### offset

The value of the `offset` field is a scalar that defines the quantity subtracted from input values during encoding. This value MUST be a JSON number.

#### scale

The value of the `scale` field is a scalar that defines the quantity multiplied with input values during encoding. This value MUST be a JSON number.

#### astype

The value of the `astype` field defines the data type of the array returned by the encoding procedure, and the data type of the array provided as input for the decoding procedure. This value MUST be Zarr V2 data type metadata.

This field was removed from the `scale-offset` codec in favor of expressing data type conversion via a dedicated codec. 

#### dtype

The value of the `dtype` field defines the data type of the array given as input to the encoding procedure, and the data type of the array returned by the decoding procedure. This value MUST be Zarr V2 data type metadata.

Note that in Zarr V3, a `dtype` field is not needed because the data type of the input to an array-array codec is determined by its location in the `codecs` metadata. Zarr V2 chunk encoding did not have such guarantees, and so the `dtype` field was necessary for the `fixedscaleoffset` codec implemented in Numcodecs.

### Algorithm

This section records a pseudocode summary of the behavior of the `numcodecs.FixedScaleOffset` codec.

#### Encoding

Given an input array `I`, an output array `O` with data type [`astype`](#astype-1) is generated by the following procedure:

```
arr = I - offset
arr = arr * scale
arr = round(arr) # round to nearest integer
O = cast(arr, astype) # Cast using NumPy casting rules
```

#### Decoding

Given an input array `I`, an output array `O` with data type [`dtype`](#dtype-1) is generated by the following procedure, which inverts the [encoding procedure](#encoding-1):

```
arr = cast(I, dtype) # Cast using NumPy casting rules
arr = arr / scale
O = arr + offset
```

### Conversion to `scale-offset`

A `numcodecs.fixedscaleoffset` codec does not correspond to a single `scale-offset` codec, but rather a sequence of codecs that includes the `scale-offset` codec. To generate this sequence of codecs, take the following steps:
- Configure a `scale-offset` codec using the `scale` and `offset` parameters from the `numcodecs.fixedscaleoffset` codec. This is the first codec in the sequence.
- Use the `astype` field to configure a codec that performs data type casting, e.g., the `astype` codec. This codec should follow the `scale-offset` codec.

#### Conversion example

This `numcodecs.fixedscaleoffset` codec:

```json
{
    "name": "numcodecs.fixedscaleoffset", 
    "configuration": {
        "astype": "uint8", 
        "scale": .1, 
        "offset": 10
        }
}
```

Would be transformed to the following codecs:

```json
[
    {
        "name": "scale-offset", 
        "configuration": {
            "scale": .1, 
            "offset": 10
            }
    },
    {
        "name": "astype", 
        "configuration": {
            "data_type": {
                "uint8"
                }
            }
    }
]
```
